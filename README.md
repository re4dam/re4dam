More specifically, my hands-on work includes .NET/ASP.NET Core with EventStoreDB and CQRS for event-driven systems, backed by RabbitMQ for message brokering and YARP as an API gateway across service boundaries. On the relational side, I work with PostgreSQL, SQL Server, and MySQL through Entity Framework Core, plus Laravel for e-commerce and API-heavy platforms. Docker keeps the whole multi-service stack reproducible. Before shipping, I write automated test cases and run functional load tests with Apache JMeter to make sure things hold up under pressure, not just in theory. On the side, C++ scratches the systems-programming itch.

**Integrations I've shipped**

- Payment & logistics APIs for Indonesian e-commerce (Midtrans, Jubelio, RajaOngkir, Shopee)

### 🌱 Currently exploring

- Data Engineering fundamentals — pipelines, storage design, and where AI-driven systems fit in
- Event sourcing as a deeper architectural pattern, beyond just the theory
- A couple of personal projects to sharpen this: a family-tree app (Next.js + GraphQL + PostgreSQL) and a small side project for managing structured character/reference data

- Linux setup troubleshooting — I've debugged stranger things than I'd like to admit
